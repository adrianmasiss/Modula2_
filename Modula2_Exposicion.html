HISTORIA Y GÉNESIS

El nacimiento de Modula-2 no fue un accidente, sino la respuesta a necesidades específicas de la ingeniería de software de los años 70, gestado en el prestigioso ETH Zürich.

📅 El Contexto: Crisis del Software (1970s)

La Problemática

A finales de los años 70, la industria del software enfrentaba lo que se llamó la "Crisis del Software". Los programas crecían en complejidad hasta volverse inmantenibles. Los lenguajes existentes (C, Fortran, Pascal puro) no proporcionaban mecanismos adecuados para estructurar programas grandes.

La programación estructurada había resuelto el problema del "código espagueti" eliminando el GOTO, pero no abordaba la organización a gran escala del software. Se necesitaba un nuevo paradigma.

Niklaus Wirth: El Arquitecto

🧑‍💻
Prof. Niklaus Wirth (1934–2024)

Prof. Niklaus Wirth (1934–2024), catedrático en el Eidgenössische Technische Hochschule Zürich (ETH Zürich), ya había creado Pascal en 1970. Sin embargo, Pascal era un lenguaje educativo para enseñar programación estructurada, no para construir sistemas operativos industriales.

La Transición: De Pascal a Modula-2

Pascal fue creado como lenguaje educativo. Sin embargo, carecía de mecanismos para compilación separada de módulos, acceso a hardware de bajo nivel y concurrencia. Wirth, en vez de "parchear" Pascal, decidió diseñar un lenguaje completamente nuevo que conservara la elegancia de Pascal pero añadiera capacidades industriales.

PASCAL (1970)
Educativo, sin módulos, sin acceso a hardware, sin compilación separada
→
MODULA-2 (1978)
Industrial, modular, acceso a bajo nivel, coroutines, compilación separada

La Motivación Directa: El Proyecto Lilith

En 1977, Wirth y su equipo en el ETH Zürich comenzaron el diseño de Lilith, una estación de trabajo personal (workstation) con interfaz gráfica, mouse y red Ethernet — años antes de que Apple lanzara la Macintosh.

Necesitaban un sistema operativo completo (Medos) y software de sistema, pero ningún lenguaje existente satisfacía sus necesidades:

✗ Pascal: Sin módulos, sin acceso a hardware
✗ C: Tipado débil, sin encapsulación formal
✗ Simula: Demasiado complejo, orientado a simulación
✗ Ada: Aún en desarrollo, demasiado pesado

Los Orígenes: Modula Experimental

Wirth comienza a experimentar con extensiones modulares para Pascal. El primer borrador, llamado simplemente "Modula", introduce el concepto de módulos pero aún mantiene muchas limitaciones de Pascal.

Innovación Clave
Separación sintáctica entre interfaz e implementación
Ubicación
Instituto de Informática, ETH Zürich, Suiza
1978

🎉 Nace Modula-2

Wirth decide no "parchear" Pascal, sino diseñar un nuevo lenguaje desde cero. Modula-2 se define formalmente como un lenguaje que introduce por primera vez el concepto de módulo (porción independiente de un programa) y encapsulación (restricción del acceso directo a componentes internos).

Del código contenido en un módulo, solo se brinda una interfaz pública definida como DEFINITION MODULE; el resto permanece oculto (encapsulado) en el IMPLEMENTATION MODULE.

primer_ejemplo.mod (1978) 
(* Primer programa escrito en Modula-2 
   por Niklaus Wirth, ETH Zürich, 1978 *)
MODULE Hello;
FROM Terminal IMPORT WriteString;
BEGIN
    WriteString("Modula-2 is born!");
END Hello.
1980–1982

Implementación en Lilith

Se completa el primer compilador de Modula-2, escrito... ¡en Modula-2! Utilizando bootstrapping desde Pascal. El sistema operativo Medos (Modular Embedded Disk Operating System) se escribe completamente en Modula-2. Esta transición de Pascal a Modula-2 fue como pasar de aprender a hacer funciones a aprender a diseñar sistemas completos.

📖 Publicación del Libro Definitivo

Wirth publica "Programming in Modula-2" (Springer-Verlag), la biblia del lenguaje. Define la sintaxis definitiva (PIM2 — Programming in Modula-2, 2nd edition). El libro se convierte en texto de referencia en universidades de todo el mundo.

1988

Estandarización ISO

Modula-2 se estandariza internacionalmente como ISO/IEC 10514-1. Se definen bibliotecas estándar y se eliminan ambigüedades del lenguaje original.

Cambios Importantes en ISO
• Separación de módulos de biblioteca en DEF/IMP
• Tipos CARDINAL e INTEGER unificados
• Bibliotecas estándar: InOut, RealInOut, Strings, etc.
1990s

Era Dorada y Competencia

Modula-2 se usa extensamente en enseñanza y sistemas embebidos. Sin embargo, C++ y Java comienzan a dominar la industria. Wirth crea Oberon (1986) como sucesor simplificado, pero Modula-2 sigue vivo en nichos industriales y educativos.

2000–2010

GNU Modula-2: El Renacimiento

Gaius Mulley comienza el proyecto GNU Modula-2 (GM2), un front-end de GCC para compilar Modula-2. El objetivo: preservar el legado y proporcionar un compilador industrial moderno, libre y multiplataforma.

2024

🎉 Integración en GCC 15

Después de 25 años de desarrollo, GM2 se integra oficialmente en la colección de compiladores GNU. Modula-2 vuelve a ser un lenguaje "de primera clase" con soporte en Linux, macOS, Windows y arquitecturas embebidas.

Dialectos Soportados
PIM2, PIM3, PIM4, ISO, M2R10 (en desarrollo)
Arquitecturas
x86_64, ARM, RISC-V, MIPS, SPARC

"Modula-2 nació de la necesidad de construir sistemas, no solo programas. Fue diseñado para que un equipo de programadores pudiera trabajar en un proyecto grande sin pisarse los unos a los otros."

— Niklaus Wirth, 1984

TIPO DE LENGUAJE

Clasificación multi-paradigma con justificación técnica

IMPERATIVO

Su modelo de ejecución está basado en la modificación explícita del estado mediante secuencias de instrucciones. El programador describe cómo hacer las cosas paso a paso.

x := 10;
y := x + 5;
WriteInt(y, 3);

ESTRUCTURADO

Utiliza estructuras de control bien definidas y bloques delimitados con BEGIN/END que evitan saltos arbitrarios. Prohibición absoluta de GOTO arbitrario.

IF cond THEN
  accion1
ELSIF cond2 THEN
  accion2
END;
CARACTERÍSTICA CLAVE

MODULAR

Permite dividir el programa en módulos independientes con separación formal entre DEFINITION MODULE (interfaz) e IMPLEMENTATION MODULE, controlando explícitamente qué se exporta y qué permanece oculto.

DEFINITION MODULE Stack;
PROCEDURE Push(c: CHAR);
END Stack.

FUERTEMENTE TIPADO

Exige declaración explícita de tipos. Prohíbe operaciones entre tipos incompatibles sin conversión explícita. Verificación estricta en tiempo de compilación.

VAR
  edad: INTEGER;
  nombre: ARRAY [0..31] OF CHAR;
(* Error: edad := nombre *)

PROCEDIMENTAL

Organización en procedimientos que encapsulan acciones específicas. Soporte para procedimientos anidados y parámetros VAR (por referencia).

PROCEDURE Swap(VAR a,b: INTEGER);
VAR temp: INTEGER;
BEGIN
  temp := a; a := b; b := temp;
END Swap;

ORIENTADO A SISTEMAS

Diseñado para desarrollo de SO, compiladores y drivers. Control sobre memoria, soporte para concurrencia (coroutines) y modularidad para sistemas complejos.

FROM SYSTEM IMPORT ADDRESS;
FROM COROUTINES IMPORT TRANSFER;

⚠️ ¿Es Modula-2 Orientado a Objetos?

No. El concepto de módulo constituye el antecedente de las clases que se observan en la POO moderna; sin embargo, la incapacidad de declarar múltiples instancias de los módulos, así como la ausencia de todo tipo de herencia, impiden afirmar que Modula-2 sea un lenguaje orientado a objetos. No obstante, se puede aproximar la herencia mediante composición usando RECORDs anidados.

simulacion_herencia.mod 
(* Simulación de Herencia con Composición *)
TYPE
    Persona = RECORD
        edad : INTEGER;
    END;

    Estudiante = RECORD
        datosPersona : Persona;  (* "hereda" de Persona *)
        carnet : INTEGER;
    END;

ciclos.mod 
(* CICLOS EN MODULA-2 *)

(* FOR - Ciclo contador *)
FOR i := 1 TO 5 DO
    WriteString("Valor de i = ");
    WriteInt(i, 3);
    WriteLn;
END;

(* WHILE - Pre-condición *)
x := 1;
WHILE x <= 3 DO
    WriteString("Valor de x = ");
    WriteInt(x, 3);
    WriteLn;
    x := x + 1;
END;

(* REPEAT - Post-condición *)
x := 1;
REPEAT
    WriteString("Valor de x = ");
    WriteInt(x, 3);
    WriteLn;
    x := x + 1;
UNTIL x > 3;

(* LOOP - Infinito con EXIT *)
LOOP
    LeerComando;
    IF cmd = 'Q' THEN EXIT; END;
    EjecutarComando;
END;

03 Procedimientos y Funciones

procedimientos.mod 
MODULE ProcedimientosDemo;
FROM InOut IMPORT WriteString, WriteInt, WriteLn;

(* Procedimiento simple sin parámetros *)
PROCEDURE Saludar;
BEGIN
    WriteString("Hola desde un procedimiento.");
    WriteLn;
END Saludar;

(* Procedimiento con parámetro por valor *)
PROCEDURE MostrarNumero(n : INTEGER);
BEGIN
    WriteString("El numero recibido es: ");
    WriteInt(n, 3);
    WriteLn;
END MostrarNumero;

(* Función con valor de retorno *)
PROCEDURE Sumar(a, b : INTEGER) : INTEGER;
BEGIN
    RETURN a + b;
END Sumar;

(* Procedimiento con VAR - por referencia *)
PROCEDURE Incrementar(VAR valor: INTEGER; cantidad: INTEGER);
BEGIN
    valor := valor + cantidad;
END Incrementar;

(* Función recursiva *)
PROCEDURE Factorial(n: INTEGER): INTEGER;
BEGIN
    IF n <= 1 THEN
        RETURN 1;
    ELSE
        RETURN n * Factorial(n - 1);
    END;
END Factorial;

VAR
    num, resultado: INTEGER;

BEGIN
    Saludar;
    MostrarNumero(25);
    WriteInt(Sumar(4, 6), 3);   (* Imprime 10 *)
    WriteLn;
    
    num := 5;
    Incrementar(num, 10);
    WriteInt(num, 5);           (* Imprime 15 *)
    WriteLn;
    
    resultado := Factorial(5);
    WriteInt(resultado, 5);     (* Imprime 120 *)
    WriteLn;
END ProcedimientosDemo.

Parámetros IN

Por valor (default). Se pasa una copia. Modificaciones no afectan al original.

PROCEDURE P(x: INTEGER)

Parámetros VAR

Por referencia. Se pasa la dirección. Modificaciones afectan al original.

Arrays abiertos

Permiten pasar arrays de cualquier tamaño.

04 Punteros y Gestión de Memoria

punteros.mod 
MODULE PunterosDemo;
FROM Storage IMPORT ALLOCATE, DEALLOCATE;
FROM InOut IMPORT WriteInt, WriteLn;

TYPE
    NodoPtr = POINTER TO Nodo;
    Nodo = RECORD
        datos: INTEGER;
        siguiente: NodoPtr;
    END;

VAR
    cabeza, actual: NodoPtr;
    i: INTEGER;

BEGIN
    (* Crear lista enlazada *)
    NEW(cabeza);
    cabeza^.datos := 1;
    cabeza^.siguiente := NIL;
    
    actual := cabeza;
    FOR i := 2 TO 5 DO
        NEW(actual^.siguiente);
        actual := actual^.siguiente;
        actual^.datos := i * 10;
        actual^.siguiente := NIL;
    END;
    
    (* Recorrer e imprimir *)
    actual := cabeza;
    WHILE actual <> NIL DO
        WriteInt(actual^.datos, 5);
        actual := actual^.siguiente;
    END;
    WriteLn;
    
    (* Liberar memoria *)
    WHILE cabeza <> NIL DO
        actual := cabeza;
        cabeza := cabeza^.siguiente;
        DISPOSE(actual);
    END;
END PunterosDemo.

⚠️ GESTIÓN MANUAL

Modula-2 no tiene garbage collector. El programador debe liberar memoria explícitamente.

NEW(ptr); (* Asignar *)
DISPOSE(ptr); (* Liberar *)

SISTEMA DE MÓDULOS

La característica distintiva de Modula-2. Separación estricta entre interfaz pública (DEFINITION) e implementación privada (IMPLEMENTATION).

DEFINITION MODULE

Define la interfaz pública. Solo contiene declaraciones, no implementación.

Stack.def 
DEFINITION MODULE Stack;

(* Interfaz pública del módulo Pila *)

CONST
    MaxStack = 100;

TYPE
    StackError = (NoError, Overflow, Underflow);

VAR
    error: StackError;

PROCEDURE Init;
PROCEDURE Push(c: CHAR);
PROCEDURE Pop(): CHAR;
PROCEDURE Full(): BOOLEAN;
PROCEDURE Empty(): BOOLEAN;

END Stack.

IMPLEMENTATION MODULE

Contiene la implementación privada. Los detalles están ocultos al usuario.

Stack.mod 
IMPLEMENTATION MODULE Stack;

VAR
    data: ARRAY [1..MaxStack] OF CHAR;
    tope: CARDINAL;

PROCEDURE Init;
BEGIN
    tope := 0;
    error := NoError;
END Init;

PROCEDURE Push(c: CHAR);
BEGIN
    IF tope < MaxStack THEN
        INC(tope);
        data[tope] := c;
        error := NoError;
    ELSE
        error := Overflow;
    END;
END Push;

PROCEDURE Pop(): CHAR;
BEGIN
    IF tope > 0 THEN
        error := NoError;
        DEC(tope);
        RETURN data[tope + 1];
    ELSE
        error := Underflow;
        RETURN CHR(0);
    END;
END Pop;

PROCEDURE Full(): BOOLEAN;
BEGIN
    RETURN tope = MaxStack;
END Full;

PROCEDURE Empty(): BOOLEAN;
BEGIN
    RETURN tope = 0;
END Empty;

END Stack.

PROGRAM MODULE (Cliente)

TestStack.mod 
MODULE TestStack;

FROM InOut IMPORT WriteString, WriteLn;
FROM Stack IMPORT Init, Push, Pop, Empty, error;

VAR
    c: CHAR;

BEGIN
    Init;
    
    WriteString("Apilando: A, B, C");
    WriteLn;
    
    Push('A');
    Push('B');
    Push('C');
    
    WriteString("Desapilando: ");
    WHILE NOT Empty() DO
        c := Pop();
        WriteString(" ");
        (* WriteChar(c); *)
    END;
    WriteLn;
    (* Resultado: C B A (LIFO) *)
END TestStack.

EL COMPILADOR

GNU Modula-2 (GM2): El compilador moderno, libre y multiplataforma integrado en GCC.

GNU Modula-2 (GM2)

GM2 es un front-end de GCC (GNU Compiler Collection) que permite compilar Modula-2 utilizando la misma infraestructura que compila C, C++, Fortran y otros lenguajes.

Desarrollado por Gaius Mulley durante más de 25 años, GM2 se integró oficialmente en GCC a partir de la versión 13, convirtiéndolo en un compilador de primera clase con soporte completo para optimizaciones avanzadas de GCC.

Genera código máquina optimizado directamente
📦

Soporta PIM2, PIM3, PIM4 e ISO Modula-2
🌐
Multiplataforma
Linux, macOS, Windows (MinGW), ARM, RISC-V
🔗

Pipeline de Compilación

01
Análisis Léxico
Tokeniza el código fuente (.mod / .def) en tokens: palabras reservadas, identificadores, literales, operadores
▼
02
Análisis Sintáctico
Construye el AST (Árbol de Sintaxis Abstracta) verificando la gramática LL(1) del lenguaje
▼
03
Análisis Semántico
Verificación de tipos, resolución de módulos importados, comprobación de visibilidad (EXPORT/IMPORT)
▼
04
Generación de GIMPLE/RTL
GM2 traduce el AST a la representación intermedia de GCC (GIMPLE), que luego se optimiza
▼
05
Código Máquina
GCC genera el ejecutable nativo para la arquitectura destino (x86_64, ARM, etc.)

EJEMPLO COMPLETO

Programa completo compilado y ejecutado con GNU Modula-2: una calculadora interactiva que demuestra módulos, procedimientos, ciclos, condicionales y E/S.

PASO 1: Definición
Calculadora.def 
DEFINITION MODULE Calculadora;

(* Interfaz pública de 
   operaciones aritméticas *)

PROCEDURE Sumar(a, b: INTEGER)
    : INTEGER;

PROCEDURE Restar(a, b: INTEGER)
    : INTEGER;

PROCEDURE Multiplicar(a, b: INTEGER)
    : INTEGER;

PROCEDURE Dividir(a, b: INTEGER;
    VAR exito: BOOLEAN) : INTEGER;

END Calculadora.
PASO 2: Implementación
Calculadora.mod 
IMPLEMENTATION MODULE Calculadora;

PROCEDURE Sumar(a, b: INTEGER)
    : INTEGER;
BEGIN
    RETURN a + b;
END Sumar;

PROCEDURE Restar(a, b: INTEGER)
    : INTEGER;
BEGIN
    RETURN a - b;
END Restar;

PROCEDURE Multiplicar(a, b: INTEGER)
    : INTEGER;
BEGIN
    RETURN a * b;
END Multiplicar;

PROCEDURE Dividir(a, b: INTEGER;
    VAR exito: BOOLEAN) : INTEGER;
BEGIN
    IF b = 0 THEN
        exito := FALSE;
        RETURN 0;
    ELSE
        exito := TRUE;
        RETURN a DIV b;
    END;
END Dividir;

END Calculadora.
PASO 3: Programa Principal
TestCalc.mod 
MODULE TestCalc;

FROM InOut IMPORT
    WriteString, WriteInt,
    WriteLn, ReadInt;
FROM Calculadora IMPORT
    Sumar, Restar,
    Multiplicar, Dividir;

VAR
    a, b, resultado : INTEGER;
    exito : BOOLEAN;

BEGIN
    WriteString("=== Calculadora ===");
    WriteLn;
    
    WriteString("Ingrese A: ");
    ReadInt(a);
    WriteString("Ingrese B: ");
    ReadInt(b);
    
    WriteString("Suma: ");
    WriteInt(Sumar(a, b), 5);
    WriteLn;
    
    WriteString("Resta: ");
    WriteInt(Restar(a, b), 5);
    WriteLn;
    
    WriteString("Multiplicacion: ");
    WriteInt(Multiplicar(a, b), 5);
    WriteLn;
    
    resultado := Dividir(a, b, exito);
    IF exito THEN
        WriteString("Division: ");
        WriteInt(resultado, 5);
    ELSE
        WriteString("Error: division/0");
    END;
    WriteLn;
    
    WriteString("=== Fin ===");
    WriteLn;
END TestCalc.
